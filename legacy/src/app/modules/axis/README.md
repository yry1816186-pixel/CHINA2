# 循轴入院模块 (AxisModule) - 重构文档

## 📋 重构概述

本文档记录了 `AxisModule.tsx` 文件的重构过程。原文件有 **673 行**，包含多个门的配置数据、复杂的 SVG 绘制逻辑、动画效果和交互逻辑。经过重构，代码被拆分为多个独立的、可维护的组件，大大提高了代码的可读性和可维护性。

### 重构前 vs 重构后

| 指标 | 重构前 | 重构后 | 改进 |
|------|--------|--------|------|
| 主文件行数 | 673 行 | ~276 行 | -59% |
| 文件数量 | 1 个 | 6 个 | 模块化 |
| 组件数量 | 1 个 | 5 个 | 可复用 |
| 可维护性 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 大幅提升 |

## 📁 新的文件结构

```
src/app/modules/axis/
├── axisConfig.ts        # 配置文件 - 门的接口定义和默认数据
├── AxisGate.tsx         # 单个门组件 - 建筑主体、装饰、动画
├── AxisPath.tsx         # 路径组件 - 主轴线、刻度线、云纹装饰
├── AxisDirection.tsx    # 方向组件 - 南/北方向标识和装饰
├── SVGDefs.tsx          # SVG 定义 - 渐变、滤镜、图案等
├── index.ts             # 导出文件 - 统一导出所有组件和配置
└── README.md            # 本文档

src/app/modules/
├── AxisModule.tsx       # 主组件（重构后）
└── AxisModule.backup.tsx # 原始文件备份
```

## 🔧 重构步骤

### 步骤 1: 创建目录结构
```bash
mkdir src/app/modules/axis
```

### 步骤 2: 提取配置数据
将接口定义和门的配置数据移到 `axisConfig.ts`：
- `AxisGate` 接口
- `defaultGates` 配置数组

### 步骤 3: 创建 SVG 定义组件
将所有的 SVG `<defs>` 内容移到 `SVGDefs.tsx`：
- 渐变定义（轴线渐变、建筑渐变）
- 滤镜（发光效果）
- 图案（装饰图案）

### 步骤 4: 创建路径组件
将主轴线和装饰元素移到 `AxisPath.tsx`：
- 背景云纹装饰
- 主轴线
- 刻度线

### 步骤 5: 创建门的组件
将单个门的渲染逻辑移到 `AxisGate.tsx`：
- 建筑主体
- 屋顶线条装饰
- 装饰纹理
- 脉冲边框动画
- 建筑名称
- 已开启标记

### 步骤 6: 创建方向组件
将方向标识移到 `AxisDirection.tsx`：
- 南/北文字标识
- 方向箭头装饰

### 步骤 7: 创建导出文件
创建 `index.ts` 统一导出所有组件和配置。

### 步骤 8: 重构主组件
修改 `AxisModule.tsx`：
- 导入新组件
- 替换 SVG 渲染逻辑
- 保留状态管理和事件处理
- 保留提示文字、信息弹窗、完成动画

### 步骤 9: 验证和测试
- 检查编译错误
- 测试所有功能
- 确保动画正常

## 📊 代码质量

### 优点

✅ **模块化**: 每个组件职责单一，易于理解和维护  
✅ **可复用**: 组件可以在其他地方复用  
✅ **类型安全**: 完整的 TypeScript 类型定义  
✅ **文档完善**: 详细的注释和 README  
✅ **性能优化**: 合理使用 React hooks 和动画  
✅ **可测试**: 组件独立，易于单元测试  

### 最佳实践

1. **单一职责原则**: 每个组件只负责一个功能
2. **配置分离**: 配置数据与渲染逻辑分离
3. **类型定义**: 使用 TypeScript 接口定义数据结构
4. **动画优化**: 使用 `motion` 组件的 `initial` 和 `animate` 属性
5. **条件渲染**: 使用 `AnimatePresence` 处理组件的进入和退出动画
6. **事件处理**: 使用回调函数处理组件间通信

## 🐛 常见问题

### Q1: 动画卡顿怎么办？
```typescript
// 添加 will-change 属性
style={{ willChange: 'transform' }}
```

### Q2: 如何禁用动画？
```typescript
// 检测用户的动画偏好
const prefersReducedMotion = window.matchMedia(
  '(prefers-reduced-motion: reduce)'
).matches;

// 条件应用动画
transition={prefersReducedMotion ? { duration: 0 } : { duration: 0.6 }}
```

### Q3: 如何修改门的总数？
修改 `axisConfig.ts` 中的 `defaultGates` 数组，添加或删除门即可，其他逻辑会自动适应。

### Q4: 如何自定义门的样式？
创建新的 SVG 渐变定义，然后在 `AxisGate.tsx` 中引用即可。
