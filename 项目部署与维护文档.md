## 5. 新增建筑案例的方法

所有新增内容统一从 `src/app/data/buildings.ts` 进入。

新增一个案例时，需要补齐以下字段：

1. `id`
2. `name`
3. `englishName`
4. `tagline`
5. `dynasty`
6. `location`
7. `category`
8. `illustration`
9. `palette`
10. `overview`
11. `significance`
12. `metrics`
13. `timeline`
14. `modules`
15. `sources`

建议新增顺序：

1. 先补 `metrics`、`timeline`、`sources`
2. 再写 `overview` 与 `significance`
3. 最后补 5 个模块的 `summary / steps / facts / takeaways`

## 6. 样式维护规则

### 允许修改的位置

- 颜色与阴影：`src/styles/theme.css`
- 字体：`src/styles/fonts.css`
- 全局基座：`src/styles/index.css`

### 不建议的做法

- 不要再新增第四套 CSS token 命名体系
- 不要在页面组件里硬编码大量全局色值
- 不要把老设计系统样式重新挂回主入口

## 7. 状态与交互维护规则

- 模块完成状态由 `getModuleProgressKey(buildingId, moduleId)` 生成。
- 不要再把模块进度只写成单独模块名，否则不同建筑会共享完成记录。
- 专题完成状态使用建筑 `id` 作为藏印唯一键。

## 8. 常见问题

### Q1：页面能构建，但样式异常

先检查：

1. `src/styles/index.css` 是否仍然引入了 `theme.css`
2. 页面是否使用了未定义的 CSS Variables
3. 是否误把旧版设计系统样式重新接回入口

### Q2：新增案例后详情页打不开

先检查：

1. `id` 是否唯一
2. `modules` 是否为空
3. `illustration` 是否属于受支持的 `IllustrationKind`

### Q3：类型检查失败

先运行：

```bash
pnpm typecheck
```

重点检查：

1. `src/app/types/index.ts`
2. `src/app/data/buildings.ts`
3. `src/app/context/ProgressContext.tsx`

## 9. 当前维护建议

1. 先在当前基线之上继续迭代，不要把旧主链重新接回入口。
2. 若准备进入第二轮治理，优先做：
   - 清理历史残留文件
   - 收缩依赖
   - 做模块级懒加载
   - 补截图回归与浏览器兼容回归
