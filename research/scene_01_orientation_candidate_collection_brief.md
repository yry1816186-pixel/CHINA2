# Scene 01 Orientation Candidate Collection Brief

## 文档定位
本文件用于给 `scene_01_orientation_order` 的下一轮 research 补录提供“候选案例 / 候选来源采集清单”。

它不是正式案例登记表，
不是正式证据主表，
也不是结论文件。

它只回答一个问题：
为了把 `scene_01_orientation_order` 从占位规划推进到 evidence-backed review readiness，接下来应该主动去找哪类案例、哪类来源、优先找什么，不该找什么。

## 当前 scene 目标重述
`scene_01_orientation_order` 服务于 `chapter_01_orientation`，目标是让用户理解：
- 建筑秩序首先通过方向、路径、阈限与层级被组织出来；
- 这是一种综合教学场景中的秩序入口，而不是单一真实完整遗址的权力路线展示；
- 视觉表达必须服从“定向与秩序可读”，而不是把 scene 拍成礼仪队列、朝会中轴或宏大对称奇观。

因此，采集对象必须优先支持：
- 方向、路径、阈限与层级关系
- 秩序如何被空间组织出来
- 哪些秩序判断可进入 scene，哪些不能被外推成礼制 / 权力 / 使用场景结论

而不是优先支持：
- 纯对称的视觉冲击
- 皇权、仪式、朝会式叙事
- 空镜中的宏大中轴感本身
- 装饰性庄严感

## 采集原则
1. 先找能支撑“秩序关系”的材料，不先找“庄重感很强”的材料。
2. 先找可复核、可映射到 `E3_supported` 以上的来源类型。
3. 先找能限制误读的来源，而不是只找能增强宏大感的来源。
4. 先形成一组足以支撑“定向与秩序入口”的最小说明材料，不追求一开始就覆盖所有路径细节。

## 优先采集的候选案例类型
### A 类：秩序关系锚点型案例
目标：支撑主秩序逻辑说明。

优先特征：
- 能帮助说明方向、路径、阈限与层级如何共同组织秩序；
- 能支撑“秩序来自空间组织”而不是仅来自表面装饰；
- 不依赖礼仪场景、人物队列或权力叙事才能成立；
- 适合被组织成 `B_composite_teaching_scene`，而不被误读成单一完整中轴实景。

筛选问题：
- 这个案例能否支持“秩序关系”而不是只支持“庄严感”？
- 这个案例会不会诱导观众把综合教学场景误读成单一真实权力路线？
- 这个案例若被综合组织，会不会因为中轴感过强而压倒路径与阈限层次？

### B 类：路径 / 阈限补强型案例
目标：用于 support 层，而不是主 anchor。

优先特征：
- 能帮助说明进入、停驻、阈限变化、层级转换；
- 适合成为 support，而不是误写成主 anchor；
- 可以帮助界定哪些路径语法可以进入 scene，哪些必须删掉。

### C 类：限制性 / 风险校正型案例
目标：不是增强“庄重感”，而是收紧误读风险。

优先特征：
- 能提醒我们哪些秩序表达仍不能被讲成礼制 / 朝会 / 权力路线；
- 能阻止“中轴奇观”“仪式通道”“帝王视角”三类滑坡；
- 能帮助界定哪些秩序判断只能保守表达。

## 优先采集的候选来源类型
### 第一优先层
- 法定文保体系中与空间组织、格局关系直接相关的正式资料
- 官方或权威资料中与路径、阈限、层级关系直接对应的部分

采集目的：
- 为定向与秩序入口提供可提升到 `E4_confirmed` 或稳固 `E3_supported` 的来源支撑

### 第二优先层
- 文保机构正式说明
- 学术论文
- 学术专著
- 教学性权威出版物

采集目的：
- 为“秩序来自方向、路径与层级组织”提供稳定支持
- 为“哪些秩序判断不能超讲”提供边界辅助

### 第三优先层
- 机构科普稿
- 展陈说明
- 可靠二手整理

采集目的：
- 只作辅助，不单独承担唯一结论
- 只在第一、第二优先层不足时补充使用

## 当前不优先采集的内容
- 只因中轴感强而显得“很有秩序”的视觉材料
- 礼仪、朝会、队列等表演性图像或叙事材料
- 只强调庄严宏大、不解释路径层级关系的传播材料
- 无法明确出处的网络整理
- 旧仓库文本、AI 文本、无来源摘录

## 采集时必须记录的字段
每个候选条目至少要记录：
- `candidate_case_name`
- `candidate_source_title_or_description`
- `source_type`
- `supports_which_claim_exactly`
- `why_this_is_anchor_or_support`
- `what_this_does_not_support`
- `initial_evidence_grade_recommendation`
- `single-site-misreading_risk`
- `ceremonial-or-power-route_risk`

## 场景级采集问题单
对每个候选案例 / 来源，都要回答：
1. 它支持的是方向 / 路径 / 阈限 / 层级中的哪一种秩序关系？
2. 它是主 anchor，还是 support，还是限制性材料？
3. 它能否帮助把当前 `E2_unresolved` 推进到 `E3_supported`？
4. 它会不会诱导我们把 orientation 讲成礼仪表演路线或权力中轴展示？
5. 它会不会让视觉团队误以为“庄严感”比“秩序可读”更重要？

## 推荐采集顺序
1. 先找 1 个秩序关系主 anchor 候选
2. 再找 1–2 个 support 来源，补路径与阈限层次说明
3. 再找 1 个限制性来源，用于收紧“礼仪 / 权力 / 中轴奇观”风险
4. 只有前三类都具备后，才讨论是否需要补更细的空间组织材料

## 当前输出目标
本轮采集不是要一次解决所有秩序问题。

本轮最低目标是：
- 让 `scene_01_orientation_order` 不再只挂在 `TBD_CASE_ANCHOR_ORIENTATION_01` 与 `SRC_TBD_ORDER_01` 上；
- 至少形成 1 个主 anchor 候选 + 1 组来源候选 + 1 个风险限制来源候选；
- 为下一轮正式回填 research 主表做好准备。

## 禁止项
- 禁止把“看起来庄严、对称”的视觉材料直接当作秩序证据。
- 禁止把 support 来源误判为主 anchor 来源。
- 禁止先扩写 packet，再回头找来源。
- 禁止为了追求庄严感，把秩序解释让位给礼仪 / 权力 / 中轴奇观叙事。
