# Reference-Grade Gap Audit

## 目的
本文件不是把当前仓库包装成“已经达到参考级”。

它只回答两个问题：
1. 当前 `web/` 执行壳离参考级作品标准还差什么；
2. 这一轮应该往前推什么，才算真正缩小差距。

## 已核对的当前事实
- 仓库当前仍明确声明：不是最终成品站，也不是全量最终视频资产库。
- `scene_03_structure_primary`、`scene_06_conservation_boundary`、`scene_01_orientation_order` 处于 `reviewed_candidate / E3_supported`。
- `scene_00_intro_entry` 仍然是 `TBD_CASE_ANCHOR_INTRO_01`，只能保守处理，不能伪装成已冻结真实入口。
- `video/11`、`video/12` 仍然不是默认执行入口。
- 现有 `web/` 页面主要完成了：scene 切换、manifest 读取、sticky stage、样片 / poster 接入占位。

## 当前页面离参考级作品标准的主要差距
### 1. 页面秩序仍偏“工具壳”而不是“作品级长滚动”
当前壳体能跑，但仍更像验证面板：
- 左侧 rail 主要是信息列表；
- 主舞台仍是单卡片结构；
- 页面节奏是功能型，不是观看型；
- scene section 的密度、停顿、张力不足。

### 2. 真实性边界还没有被前景化
当前 manifest 中已经有：
- `status`
- `anchor`
- `sourceRef`
- `allowedMode`

但这些信息主要停留在元数据层，
没有成为观众观看时持续可见的“约束语言”。

结果是：
- user 知道这是受控候选；
- 页面却还没有把“受控候选”转成第一视觉层。

### 3. `scene_00_intro_entry` 的风险提示不够强
虽然主表和 manifest 已经明确 intro 未冻结，
但当前页面还没有把这种风险持续、显式地钉在入口结构里。

对于参考级长滚动页面来说，
入口不是一句备注，
而是整套观看协议的起点。

### 4. 页面缺少统一的场景纪律面板
当前 scene 的“不得变成什么”主要存在于 docs / packet / notes 中。
页面层尚未把以下内容形成统一观看面板：
- allowed mode
- evidence grade
- must-not-become
- reading task
- anchor scope

这会让页面更像“场景展示”，而不是“受控阅读系统”。

### 5. 当前壳体没有形成更接近最终作品的艺术方向样机
现有壳体完成了运行层的最小目标，
但还没有给出一个更接近最终作品的可审样机，
因此“下一步该收口成什么气质”仍不够具体。

## 这一轮应做、而且已经开始做的事
本轮不继续堆抽象规范，
而是直接补一个并行的 `web/reference-shell/`：

- 不替换现有执行壳；
- 不假装已经 final；
- 但把页面组织推进到更接近参考级作品的结构样机。

### 本轮推进重点
1. 把“观看协议”从备注提升为页面首层结构
2. 把真实性边界从 metadata 提升为持续可见的约束面板
3. 把 scene section 从普通说明块推进为更有节奏的长滚动章节单元
4. 把 intro unresolved 风险做成强披露，而不是弱提示
5. 给 first-wave 提供一个更接近成品气质的可审外壳

## 本轮仍然没有解决的事
这一轮即使做完，也**不能**声称已经解决：
- 最终网页动效系统
- 最终播放器行为
- 最终字体系统与微交互
- 最终整站信息架构
- `scene_00_intro_entry` 的真实 anchor 定稿
- 参考站级别的完成度验证

## 关于参考网站
当前环境不能实际浏览外部参考网站，
因此本文件不声称“已经对照参考站逐项完成”。

当前只能做的是：
- 先把仓库内部结构与页面样机推进到更接近参考级的形态；
- 等环境支持外部浏览后，再做真正的参考对照审查。

## 当前结论
CHINA2 目前已经有：
- 较强的主表与 research 约束；
- first-wave 的受控候选基础；
- 一个可运行的前端执行壳。

但它距离“参考级作品网页”仍差一层关键跨越：
**从执行壳，跨到具有观看协议、审美收口、真实性披露和场景纪律前景化的作品级页面样机。**

`web/reference-shell/` 的作用，就是完成这次跨越的第一步。
