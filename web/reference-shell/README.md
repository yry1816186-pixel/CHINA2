# Reference Shell

## 定位
这是一个**并行存在**的页面样机目录。

它不是对现有 `web/` 执行壳的否定，
也不是最终上线站。

它的作用只有一个：
把 CHINA2 从“能跑的 first-wave 执行壳”，推进到“更接近参考级作品标准的页面结构样机”。

## 为什么不直接覆盖旧壳
因为当前仓库还没有资格把任何页面假装成 final：
- `scene_00_intro_entry` 尚未冻结
- first-wave 仍主要是 `reviewed_candidate`
- 最终播放器、最终动效、最终艺术系统都未定稿

因此本目录采用并行策略：
- 旧 `web/` 保留最低执行基线
- 新 `reference-shell/` 负责验证更接近作品级页面的观看秩序

## 这个目录解决什么
- 把真实性边界前景化
- 把 scene 纪律做成持续可见的页面层
- 把 intro 未冻结风险做成强披露
- 把 long-scroll 的停顿、推进、切换做得更像作品级章节结构

## 这个目录不解决什么
- 不解决最终全站工程化
- 不解决最终播放器系统
- 不解决最终字体与动效定稿
- 不宣称已经达到参考站完成度

## 运行方式
在 `web/` 目录启动静态服务器后，访问：

- `http://localhost:8000/reference-shell/`

页面会读取上一级的：
- `../scenes.manifest.json`

因此它与当前主表接入方式兼容，
不会另起一套事实源。

## 审查重点
打开页面后，请重点看：
1. unresolved intro 是否被强披露
2. active scene 的 status / anchor / evidence / mode 是否成为首层观看语言
3. 章节滚动是否比旧壳更有停顿、推进与秩序
4. 三条样片接入后，页面气质是否更接近同一作品系统

## 当前结论
本目录是一次**向参考级作品标准逼近的结构性推进**，
不是“项目已经完成”的证明。
