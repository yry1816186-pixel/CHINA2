# 02 Narrative Blueprint

## 叙事目的
本文件用于冻结全站教学逻辑、总叙事结构与章节排序原则，确保空间游历是组织方法，礼制/结构/工艺/保护是知识嵌套层，而不是并列分支。

## 受众与体验目标摘要
- 受众优先级：大众优先，专业用户为次级观察者。
- 体验目标顺序：先进入，再理解，再复述。
- 页面必须保证首层可进入、二层可展开、三层可追溯。

## 学习结果目标摘要
项目完成后，用户至少应能：
1. 说明建筑如何通过空间组织秩序。
2. 理解结构逻辑不是单独漂浮于外观之外。
3. 识别工艺细部服务于整体营造系统。
4. 区分历史阅读层与当代保护层。

## 五段总叙事结构
| `segment_id` | 段落功能 | 说明 |
| --- | --- | --- |
| `segment_00_invocation` | 建立观看关系 | 说明如何进入本项目、如何观看综合场景 |
| `segment_01_order_entry` | 进入秩序系统 | 建立方向、路径、等级与空间收束 |
| `segment_02_structural_climax` | 达到空间与结构高潮 | 将最高解释密度放在结构阅读层 |
| `segment_03_return_to_detail_and_care` | 回落到工艺与保护 | 从高潮回落到细部与保护责任 |
| `segment_04_controlled_exit` | 完成总结与离场 | 不新增知识分支，只收束与回望 |

## 六章映射逻辑
`chapter_00_intro` 为序章入口，不计入六章主章逻辑。六章主章为：
1. `chapter_01_orientation`
2. `chapter_02_progression`
3. `chapter_03_structure`
4. `chapter_04_craft`
5. `chapter_05_conservation`
6. `chapter_06_outro`

映射原则：
- `segment_01_order_entry` 对应 `chapter_01_orientation` 与 `chapter_02_progression`
- `segment_02_structural_climax` 对应 `chapter_03_structure`
- `segment_03_return_to_detail_and_care` 对应 `chapter_04_craft` 与 `chapter_05_conservation`
- `segment_04_controlled_exit` 对应 `chapter_06_outro`

## 嵌套层进入顺序
1. 先以空间游历建立观看关系。
2. 再以礼制和路径进入秩序层。
3. 再进入结构理解的最高密度章节。
4. 再把工艺放回整体系统中阅读。
5. 最后以保护层完成当代回望。

## 不可偏离的叙事边界
- “空间游历”是承载方式，不是新主题。
- “保护”是回望层，不是与礼制、结构、工艺并列开启的平行入口。
- “综合场景”用于教学组织，不得冒充单一真实遗址。
- “高潮”必须承载结构理解，不得仅承载视觉震撼。

## 确定项
- 五段结构固定。
- `chapter_03_structure` 为解释密度最高的结构高潮章。
- `chapter_05_conservation` 为当代保护回望章，而非新主题分叉。
- `chapter_06_outro` 只负责收束，不开启新知识分支。

## 待验证项
- 各段占比和章节负载预算需在后续章节规格与评审表中量化。
- 各章最终采用的案例支撑比例待验证。

## 禁止项
- 禁止把内容重排为朝代通史。
- 禁止把章节重排为并列图鉴。
- 禁止把高潮章做成纯氛围视频章。
- 禁止在尾章新增核心知识分支。
