# 04 Responsibility Matrix

## 职责目的
本文件用于冻结视频、交互、程序动画三类媒介的职责边界、失败测试和越权判定，避免把最难修改的媒介承担最不稳定的知识内容。

## 媒介职责矩阵
| 信息类型 | `canonical_owner` | 可辅助媒介 | 不可单独承担的媒介 |
| --- | --- | --- | --- |
| 场景进入、离场氛围 | `video` | `2.5D`, `vector_programmatic` | `static_reference` |
| 路径、层级、方向关系 | `vector_programmatic` | `2.5D`, `video` | `video` |
| 结构剖示、比较关系 | `vector_programmatic` | `2.5D` | `video` |
| 章节状态与用户反馈 | `vector_programmatic` | 无 | `video`, `static_reference` |
| 证据锚点与说明引用 | `static_reference` | `vector_programmatic` | `video` |
| 氛围补充与节奏连接 | `video` | `2.5D` | `static_reference` |

## Canonical owner rule
- 每一条核心知识点必须指定唯一 `canonical_owner`。
- 辅助媒介可以帮助进入或收束，但不得独立改写主解释。
- 若多媒介同时承载同一知识点，以 `canonical_owner` 为唯一正确版本。

## Failure tests
### `去视频测试`
- 目的：验证核心事实是否被错误地塞进视频。
- 失败条件：去掉视频后章节无法理解 `core_claim`。

### `去交互测试`
- 目的：验证基础理解是否被错误地塞进深度交互。
- 失败条件：无交互时用户无法理解章节最基本结论。

### `去程序动画测试`
- 目的：验证程序动画是否越权补事实。
- 失败条件：去掉程序动画后，事实层缺失或关系逻辑被改写。

## 越权判定
- 视频越权：当 `video` 成为某章节唯一事实解释层。
- 交互越权：当交互为“好玩”而新增未被规范批准的信息关系。
- 程序动画越权：当程序层制造未经证据批准的结构、礼制或保护结论。

## 审核角色与阻断权
| 角色 | 职责 | 阻断权 |
| --- | --- | --- |
| `spec_owner` | 规范总控 | 对冻结项拥有最终阻断权 |
| `historical_reviewer` | 史实审校 | 对事实漂移拥有优先阻断权 |
| `technical_validator` | 技术验证 | 对不可实现或不可降级内容拥有阻断权 |
| `aesthetic_reviewer` | 审美审校 | 对风格失控与误导性表现拥有阻断权 |

阻断优先级：`historical_reviewer` 与 `spec_owner` 为最高优先级；任何带病推进必须记录并重新审查。

## 确定项
- 核心事实不得只活在视频里。
- 可比较关系不得只 baked 进渲染媒体。
- 程序动画不得创造事实。
- 去除测试为强制验收动作，不是建议项。

## 待验证项
- 各章具体媒介占比待在章节规格和资产清单中细化。
- 审核角色的组织名称可调整，但职责和阻断顺位不可调整。

## 禁止项
- 禁止用视频代替全部解释层。
- 禁止用程序动画伪装复原结论。
- 禁止用深度交互掩盖章节逻辑缺失。
