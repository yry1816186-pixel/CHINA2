# 11 Heavy Reference File Usage

## Purpose
This file explains how to use the heavy reference files in the current phase.

## Files covered
- `video/11_full_generation_plan.md`
- `video/12_full_prompt_packets_master.md`
- `video/13_generation_sequence_and_priority.md`
- `video/14_scene_exclusion_and_downgrade_list.md`

## Current usage pattern
In the current phase, these files are best used as reference files.

They are useful for:
- broad package review
- cross-checking whole-scene coverage
- comparing priority and downgrade logic
- reviewing already-expanded planning text

They are not the recommended default entry for normal day-to-day work.

## Recommended working order
Start with:
- `docs/09_canonical_working_entry.md`
- `docs/08_active_context_profile.md`
- `video/16_execution_status_and_activation.md`
- `video/17_priority_scene_execution_profile.md`

Then use the heavier reference files only when needed.

## Practical rule
If a task can be completed without loading the heavy reference files, do not load them by default.
