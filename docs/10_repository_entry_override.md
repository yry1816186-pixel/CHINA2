# 10 Repository Entry Override

## Purpose
This file clarifies the current recommended repository entry point for the project.

It is intended to be read together with:
- `docs/08_active_context_profile.md`
- `docs/09_canonical_working_entry.md`
- `video/16_execution_status_and_activation.md`

## Current phase
The repository should currently be understood as:
- a frozen planning baseline
- a controlled video planning workspace
- a research-hardening workspace for priority scenes

It should not currently be treated as:
- a front-end implementation repo
- a keyframe execution repo
- a batch external video generation execution repo

## Recommended entry order
1. `docs/09_canonical_working_entry.md`
2. `docs/08_active_context_profile.md`
3. `video/16_execution_status_and_activation.md`
4. `research/priority_evidence_closure_plan.md`

Then proceed to the minimal active-context layer.

## Practical note
When older entry wording and the newer control documents seem to point in different directions, prefer the newer control documents for day-to-day work in the current phase.
