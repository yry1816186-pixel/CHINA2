# New Chat Takeover Brief

## 这份文件的用途
当当前对话已经变长、注意力可能漂移、或者需要在新窗口中继续推进 CHINA2 项目时，必须先阅读本文件。

它的作用不是总结漂亮话，
而是防止新窗口 assistant：
- 脑补项目成熟度
- 误判当前仓库已经“最终定稿”
- 自由发挥改写项目目标
- 忽略已有主表与 scene 状态
- 把未冻结入口误做成单一真实遗址 hero scene

## 项目真正目标
目标不是做一个“古风网页”，也不是做一个“宣传型知识专题”。

目标是：
- 做出一个作品级、艺术级、工艺级的中国古代建筑长滚动交互网页
- 同时具备知识秩序、观看秩序、真实性边界和高级审美控制
- 最终效果参考用户明确给出的高标准长滚动作品级页面，而不是一般专题页或一般产品页

## 当前必须承认的现实状态
### 已完成到较强状态的部分
- 规范冻结、上下文减重、工作入口重排
- first-wave 的主表回填与 candidate review 工作
- `scene_03_structure_primary`、`scene_06_conservation_boundary`、`scene_01_orientation_order` 已进入 `reviewed_candidate`
- 已存在 `web/` 执行壳
- 已存在可直接复制的视频文本文件 `web/DIRECT_COPY_VIDEO_TEXTS.md`

### 仍然没有完成的部分
- 项目不是最终成品站
- 项目不是最终全量视频资产库
- `scene_00_intro_entry` 仍未进入主表真实 anchor 注册
- 真实网页的最终视觉系统、最终交互动效、最终播放器行为还没有定稿
- 整站是否真的达到参考级作品质量，当前还不能诚实地宣称“已经保证”

## 当前状态的硬判断

- `chapter_01_orientation` 当前候选是 `CANDIDATE_FORBIDDEN_CITY_ORIENTATION_01`
- `chapter_03_structure` 当前候选是 `CANDIDATE_FOGUANG_EAST_HALL_01`
- `chapter_05_conservation` 当前候选是 `CANDIDATE_PINGYAO_CONSERVATION_01`
- 上述候选都只是 `reviewed_candidate / E3_supported`，不是最终冻结事实层

## 当前最容易出现的错误
1. 把当前仓库误判为“已经可以直接完美产出最终作品”
2. 把 `scene_00_intro_entry` 做成单一真实遗址入口
3. 把 `scene_01_orientation_order` 做成礼仪路线、中轴奇观或权力叙事
4. 把 `scene_03_structure_primary` 做成技术奇观或纪录式实景证明
5. 把 `scene_06_conservation_boundary` 做成旅游宣传片或修复奇迹片
6. 让 agent 自由发挥，跳出仓库现有约束体系
7. 误把 `web/` 执行壳当作最终成品站

## 新窗口 assistant 的工作原则
- 绝不脑补“已经完美”
- 绝不把当前状态说成已经全部妥善
- 绝不为了安抚用户而夸大成熟度
- 绝不自由改写项目目标
- 绝不回到“文件很多所以很专业”的假成熟状态
- 每一步都必须明确：当前是主表候选、受限执行，还是最终冻结

## 关于外部参考网站
用户明确给出了高标准参考网站。
新窗口 assistant 必须：
- 如果环境支持浏览，先实际查看这些参考网站的页面节奏、密度、动画、留白、信息结构，再给判断
- 如果环境不支持浏览，必须明确说“当前无法实际查看参考站点”，不能假装已经看过

## 当前最正确的推进方向
如果继续推进，不应再主要增加抽象规范。
应优先推进：
1. 页面最终艺术方向收口
2. agent 的唯一实现边界收口
3. `scene_00_intro_entry` 的最终策略定稿
4. `web/` 从执行壳向更接近成品的结构继续推进

## 给新窗口 assistant 的一句话
不要安抚用户。
不要夸大成熟度。
不要装作已经完美。
请把 CHINA2 当成一个仍在冲刺、但已经有较强基础的作品级项目，继续做严肃、克制、证据导向、可落地的总控推进。
