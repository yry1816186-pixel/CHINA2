# 项目全量诊断报告

## 1. 诊断结论

本次诊断日期：2026-03-24  
项目当前结论：**P0 阻塞问题已解除，主运行链已重建并通过构建与类型检查。**

整改前，项目存在以下核心问题：

1. 构建链路损坏，`pnpm build` 直接失败，主入口无法稳定交付。
2. 活跃页面、入口文案与部分数据文件存在大面积乱码，内容不可读。
3. 数据模型、页面契约、进度状态逻辑同时存在新旧两套实现，造成维护失控。
4. 样式系统分裂，`index.css`、`theme.css`、设计系统遗留样式并存，token 命名不一致。
5. 工程层缺失 `tsconfig` 与 `typecheck`，类型层分叉问题无法在提交前被发现。
6. 仓库仍保留较多历史备份、旧模块与测试残留，容易干扰后续治理判断。

## 2. 分维度问题清单

### 2.1 内容逻辑

- 等级：严重
- 影响范围：首页、详情页、旧数据源
- 主要表现：
  - 叙事结构不稳定，不同建筑被错误套入同一模块玩法。
  - 同时存在 `src/app/data.ts` 与 `src/app/data/buildings.ts` 两套内容入口。
  - 部分内容出现乱码，无法形成有效文化表达。

### 2.2 视觉设计

- 等级：严重
- 影响范围：首页、详情页、错误页、全局样式
- 主要表现：
  - 样式 token 有多套命名体系并行存在。
  - 页面虽有视觉效果，但主题表达与运行时变量不统一。
  - 错误页仍引用旧变量，异常态存在样式失真风险。

### 2.3 交互动效

- 等级：一般到严重
- 影响范围：模块可视化、进度状态
- 主要表现：
  - 旧模块残留较多，交互逻辑与内容结构脱节。
  - 模块完成状态曾存在跨建筑串联问题。

### 2.4 技术架构

- 等级：致命
- 影响范围：入口、数据、上下文、页面层
- 主要表现：
  - `types`、`data`、`context` 三层契约分叉。
  - 缺少 TypeScript 校验基座。
  - 活跃链路与历史链路混杂，入口判断成本高。

### 2.5 性能与兼容性

- 等级：一般
- 影响范围：构建产物、依赖面
- 主要表现：
  - 依赖面明显膨胀，存在较多当前主链未使用依赖。
  - 仓库保留旧 `dist` 产物与备份文件，影响交付边界清晰度。

### 2.6 文化准确性

- 等级：严重
- 影响范围：建筑介绍、时间线、专题叙事
- 主要表现：
  - 旧数据源中部分内容来源混乱、文本失真。
  - 案例之间缺少针对不同建筑类型的差异化叙述。

## 3. 根因分析

1. 项目曾经历多轮“叠加式修改”，没有统一的数据契约与组件边界。
2. 页面直接硬编码内容与状态逻辑，导致一处改动无法全局收敛。
3. 缺少类型检查、编码治理与交付流程约束，历史问题持续扩散。
4. 遗留文件未归档，造成主链与废弃链路长期共存。

## 4. 本轮整改动作

### 4.1 已完成

- 重建主入口路由，仅保留首页与建筑详情页两条稳定主链。
- 重写 `src/app/types/index.ts`，统一建筑、模块、指标、时间线与进度类型。
- 重建 `src/app/data/buildings.ts`，收敛为单一结构化内容源。
- 重写 `ProgressContext`，修复“不同建筑共用模块完成状态”的逻辑缺陷。
- 重建首页、详情页、建筑卡片、模块可视化组件。
- 修复 `ErrorBoundary`、`index.html`、全局样式基座乱码与 token 不一致问题。
- 新增 `tsconfig.json`、`typecheck` 脚本，并补齐 TypeScript 类型依赖。
